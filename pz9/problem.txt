Задание: 

Описать класс согласно варианту. В классе должны быть конструкторы, деструктор, 
перегруженные операции << и >>. При необходимости и/или по желанию можно добавить другие методы.

Описать класс «хеш-таблица». В хеш-таблице разрешение коллизий происходит с помощью метода цепочек, 
используется модульная хеш-функция. Данная хеш-таблица работает с объектами созданного класса. 
Ключ указан в варианте. В хеш-таблице не могут находится два и более объекта с одним и тем же значением ключа.
При определении класса «хеш-таблица» использовать неупорядоченный контейнер unordered_set<>
В хеш-таблице должны выполняться следующие действия: 
- добавление элемента в хеш-таблицу (по ключу);
- удаление элемента из хеш-таблицы (по ключу);
- поиск элемента по ключу в хеш-таблице;
- вывод всех элементов хеш-таблицы.
При вставке объектов класса в хеш-таблицу использовать либо ввод с клавиатуры, либо ввод из файла.
Можно добавить другие методы.
Оттестировать все методы класса хеш-таблица с помощью тестирующего проекта.
Написать программу, демонстрирующую работу с этим классом. 

Вариант 2:
Класс сельскохозяйственный продукт содержит поля: код продукта, наименование 
района (где выращивают), наименование продукта, площадь (га), урожайность (кг/га),
цена за 1 кг, потери при транспортировке (%), стоимость продукта.
Ключ – код продукта.
